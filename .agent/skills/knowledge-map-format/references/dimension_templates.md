## DIM-7: Pitfalls（踩坑记录）

**固定格式，每个坑的结构严格一致：**

```
# {Topic} 踩坑记录

> ⚠️ **这是知识库中最有价值的维度。** 每次踩坑后请追加条目。

---


## 坑 N: {坑标题}

**场景：** {什么情况下会遇到}

**症状：** {用户看到的现象}

**根因：** {为什么会发生}

**解法：**

❌ 错误写法 — 原因

    错误代码（4 空格缩进，绝不嵌套代码块）

✅ 正确写法 — 原因

    正确代码（4 空格缩进）

**教训：** {一句话总结}

> 📖 Docs: [来源](URL) — 章节

---

（每个坑之间 `---` 分隔，格式完全一致）


## 调试清单

1. [ ] **检查项 1？** → 操作建议
2. [ ] **检查项 2？** → 操作建议
...
```

**关键格式规则：**
- ✅ 用 **加粗关键词**（场景/症状/根因/解法/教训）+ 同行内容
- ❌ 不用 `###` 三级标题
- ✅ 解法用 ❌/✅ + 4 空格缩进
- ❌ 绝不在 ` ```markdown ` 中嵌套 ` ```bash `
- ✅ 每个坑结尾有 `> 📖` 引证
- ✅ 调试清单中的检查项用 **加粗**

> 📖 参考: `claude_code_skill_pitfalls.md` — 10 坑 + 调试清单

## 质量检查清单

生成完所有文件后，逐项检查：

- [ ] **R1 Frontmatter** — 每个文件都有完整 frontmatter（6 个必填字段）
- [ ] **R2 引证** — 每个 `##` 章节结尾都有 `>` 引证块
- [ ] **R3 无嵌套** — 没有在 ` ```markdown ` 中嵌套代码块
- [ ] **R4 对比格式** — ❌/✅ 用缩进，不用代码块包裹
- [ ] **R5 分隔线** — `##` 之间有 `---`，`###` 之间没有
- [ ] **R6 标题引证** — `# 标题` 下紧跟全局引证行
- [ ] **章节编号** — Map 7章 / Concepts 5章 / Math 5章 / Tutorial 6 Section / Usage 场景+速查 / Code 4章 / Pitfalls 坑+清单 / History Station / Bridge 6表
- [ ] **关键词格式** — Pitfalls 用 **加粗关键词**，History 用 **问题/创新/局限**
- [ ] **表格一致** — 术语/辨析/速查/文件地图都用 Markdown 表格
- [ ] **Box-drawing** — 架构图用 `┌───┤` 风格，不用 Mermaid
- [ ] **文件命名** — `{topic}_{dimension}.md`
